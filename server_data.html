Тестовое задание (Developer)

Задания, описанные ниже не сложные, но каждое их них может иметь некоторые подводные камни, обойти которые поможет умение разбираться в документации и решать возникающие проблемы. Задания связаны между собой логически, но если какое-либо задание сделать по каким-либо причинам не удается, можно приступать к следующему, задав ему необходимые входные данные. Скрипты должны работать на PHP 5.2.

Задача #1

Используя curl (http://php.net/manual/en/book.curl.php), получить контент с данной страницы (http://www.cbonds.info/sandbox/some_source.php) и записать его в файл (server_data.html)

Задача #2

Используя регулярные выражения PCRE (http://php.net/manual/en/book.pcre.php), разобрать данные из таблицы (Some table) файла server_data.html и сформировать из них xml-файл (emissions.xml) следующего формата: 

<?xml version="1.0" encoding="utf-8"?>
<items>
	<item>
		<property title="Эмитент"><![CDATA[Открытое акционерное общество "АМУРМЕТАЛЛ"]]></property>
		<property title="Эмитент (кратко)"><![CDATA[ТГК-2]]></property>
		<!-- ... для каждого свойства в таблице -->
	</item>
	<item>
		<property title="Эмитент"><![CDATA[Открытое акционерное общество "Нефтяная компания "ЛУКОЙЛ""]]></property>
		<property title="Эмитент (кратко)"><![CDATA[ЛУКОЙЛ]]></property>
		<!-- ... для каждого свойства в таблице -->
	</item>
	<!-- ... для каждой строки в таблице -->
<items>
Для формирования xml-файла должен использоваться DOM (http://php.net/manual/en/book.dom.php).

Задача #3

Написать xsl-шаблон style.xsl для вывода данных из emissions.xml в следующем формате:
Эмитент: Открытое акционерное общество "АМУРМЕТАЛЛ"
Эмитент (кратко): ТГК-2
Тип: Облигация ООО
Выпуск: Вып. 1
... для каждого свойства из xml
Открытое акционерное общество "Нефтяная компания "ЛУКОЙЛ""
Эмитент (кратко): ТГК-2
Тип: Облигация ООО
Выпуск: Вып. 1
... для каждого свойства из xml
... для каждого элемента из xml своя "карточка" ...

Использовать конструкции xsl:if, xsl:choose, xsl:for-each, xsl:call-template крайне не рекомендуется.

Трансформация должна происходить на сервере с использованием xsltProcessor (http://php.net/manual/en/class.xsltprocessor.php).
Скрипт для осуществления трансформации можно взять прямо из примеров в документации http://php.net/manual/en/xsltprocessor.construct.php.

Задача #4

Предположим, что данные из таблицы "some table" требуется сохранять в БД: какой на ваш взгляд должна быть ее структура?
Результатом должен быть sql_dump БД, структура таблиц которой реализует возможность хранения данных в нормализованном виде.

Задача #5

Составить запрос, который бы агрегировал данные из таблиц задания #4 по столбцу "Тип", суммиря столбец "номинал". Записи неопределенного типа также должны войти в выборку. Сортировка должна вестись по убыванию суммы номиналов.
Итоговая выборка в запросе должна содержать поля:
Тип | Номинал | Список эмитентов данного типа (перечисленных череp запятую).
